# 화면 플로우

이 문서는 조과로그 1차 MVP의 핵심 화면 이동 흐름을 정의한다.

1차 MVP의 화면 흐름은 두 가지를 중심으로 한다.

- 내 조과 목록에서 조과를 직접 등록하는 흐름
- AI 어종 판별기에서 사진으로 어종을 판별한 뒤 조과 등록으로 이어지는 흐름
- 잡은 어종 도감을 확인하는 흐름

## 전체 탭 구조

하단 탭은 1차 MVP에서 아래 구조를 기본으로 한다.

```text
홈
  -> 내 조과 목록

조과 등록
  -> step1: 민물/바다 선택
  -> step2: 어종 정보 등록
  -> step3: 지도 기반 위치 등록

AI 어종 판별기
  -> 이미지 선택/촬영
  -> 어종 판별
  -> 어종 정보 표시
  -> 조과 등록으로 이동

도감
  -> 잡은 어종 목록
  -> 어종 일러스트 확인
  -> 어종 상세
```

탭 이름과 실제 route 이름은 구현 중 조정할 수 있지만, 사용자 흐름은 유지한다.

## 플로우 1: 직접 조과 등록

사용자 흐름:

```text
로그인
  -> 홈
  -> 내 조과 목록
  -> 하단 탭의 조과 등록
  -> step1: 민물/바다 선택
  -> step2: 어종 정보 등록
  -> step3: 지도 기반 위치 등록
  -> 조과 등록 완료
  -> 내 조과 목록 또는 조과 상세
```

화면별 역할:

- 로그인: 카카오, 애플, 구글 로그인 제공
- 홈: 내 조과 목록을 보여준다.
- 조과 등록 step1: 사용자가 민물/바다를 선택한다.
- 조과 등록 step2: 어종, 마릿수, 물때, 날씨, 이미지, 메모 등 조과 정보를 입력한다.
- 조과 등록 step3: 카카오지도 기반으로 위치를 선택하고 위치 이름, 위도, 경도를 저장한다.
- 등록 완료 후: 사용자가 방금 등록한 기록을 확인할 수 있어야 한다.

권장 route 구조:

```text
app/(tabs)/index.tsx
app/(tabs)/catch-log/index.tsx
app/catch-log/write/step1.tsx
app/catch-log/write/step2.tsx
app/catch-log/write/step3.tsx
app/catch-log/[id].tsx
```

구현 중 route를 다르게 잡아도 되지만, step 흐름은 명확해야 한다.

## 플로우 2: AI 어종 판별 후 조과 등록

사용자 흐름:

```text
로그인
  -> 홈
  -> 내 조과 목록
  -> 하단 탭의 AI 어종 판별기
  -> 사진 촬영 또는 폴더에서 이미지 업로드
  -> 어종 판별 요청
  -> 어종 정보 표시
  -> 조과 등록으로 이동
  -> 조과 등록 step1/step2/step3 중 필요한 단계로 이어짐
  -> 조과 등록 완료
```

화면별 역할:

- AI 어종 판별기: 사진 촬영 또는 이미지 업로드 진입점
- 이미지 선택/촬영: `expo-image-picker`를 사용한다.
- 어종 판별 요청: 이미지를 Supabase Storage 또는 Edge Function 입력으로 전달한다.
- 어종 정보 표시: Gemini 3.1 AI 판별 결과를 후보 형태로 보여준다.
- 조과 등록 이동: 사용자가 선택한 어종 후보를 조과 등록 폼의 초기값으로 넘긴다.

권장 route 구조:

```text
app/(tabs)/ai-species/index.tsx
app/ai-species/result.tsx
app/catch-log/write/step1.tsx
app/catch-log/write/step2.tsx
app/catch-log/write/step3.tsx
```

## 플로우 3: 잡은 어종 도감 확인

사용자 흐름:

```text
로그인
  -> 홈
  -> 하단 탭의 도감
  -> 잡은 어종 목록 확인
  -> 잡은 어종의 일러스트 이미지 확인
  -> 어종 상세 확인
  -> 해당 어종의 내 조과 목록 또는 최근 조과로 이동
```

화면별 역할:

- 도감 목록: 사용자가 잡은 어종을 도감 형태로 보여준다.
- 잠긴 어종: 아직 잡지 못한 어종은 잠김 상태로 보여준다.
- 어종 상세: 어종 일러스트, 이름, 내가 잡은 횟수, 최근 조과, 최대 사이즈 등을 보여준다.

권장 route 구조:

```text
app/(tabs)/species-dex/index.tsx
app/species-dex/[id].tsx
```

1차 MVP에서는 전체 어종 백과가 아니라 "내가 잡은 어종 도감"을 우선한다.

## 조과 등록 step 정의

### Step 1: 민물/바다 선택

목적:

- 조과가 민물 낚시인지 바다 낚시인지 먼저 구분한다.
- 이후 어종 후보, 입력 필드, 통계 분류에 영향을 줄 수 있다.

저장 후보:

- `waterType`: `"freshwater"` 또는 `"saltwater"`

UX 기준:

- 선택지는 크고 명확해야 한다.
- 뒤로 가도 선택값이 유지되어야 한다.

### Step 2: 어종 정보 등록

목적:

- 조과의 핵심 정보를 입력한다.

입력 후보:

- 어종
- 마릿수
- 물때
- 날씨
- 이미지
- 메모
- AI 판별 결과에서 넘어온 어종 후보

UX 기준:

- AI 판별 결과는 자동 확정하지 않는다.
- 사용자가 어종을 직접 수정할 수 있어야 한다.
- 필수값과 선택값을 명확히 구분한다.
- 이미지 업로드 실패 시 다시 시도할 수 있어야 한다.

### Step 3: 지도 기반 위치 등록

목적:

- 카카오지도 기반으로 출조 위치를 기록한다.

저장 후보:

- 위치 이름
- 주소
- 위도
- 경도

UX 기준:

- 현재 위치 또는 검색 기반 위치 선택을 고려한다.
- 위치 권한이 거부된 경우에도 직접 검색/선택할 수 있어야 한다.
- 등록 완료 전 최종 확인이 가능하면 좋다.

## 상태 전달 규칙

조과 등록 step 간 상태는 화면 이동 중 사라지면 안 된다.

권장 방식:

- 단순 MVP에서는 상위 write layout 또는 전용 hook에서 임시 form state를 관리한다.
- 복잡해지면 React Hook Form의 `FormProvider` 범위를 step 전체로 잡는다.
- 서버 저장은 마지막 step에서 한 번에 처리하는 방식을 우선 검토한다.

주의:

- route params에 큰 이미지 객체나 전체 form 데이터를 넣지 않는다.
- AI 판별 결과는 필요한 최소 식별자나 간단한 초기값만 전달한다.
- 이미지 파일은 업로드 전 local uri, 업로드 후 storage path를 구분한다.

## AI 판별 결과 처리 규칙

- AI 판별 결과는 후보로 보여준다.
- 가장 가능성이 높은 후보를 기본 선택으로 둘 수 있지만 자동 확정하지 않는다.
- 사용자는 후보를 바꾸거나 직접 입력할 수 있어야 한다.
- 조과 등록 저장 시에는 `ai_species_predictions`와 사용자가 최종 선택한 어종을 구분해서 저장한다.
- AI 판별 실패 시 조과 등록 자체를 막지 않는다. 사용자가 직접 어종을 입력할 수 있어야 한다.

## 도감 처리 규칙

- 도감은 사용자가 최종 선택해 저장한 어종을 기준으로 해금한다.
- AI 판별 후보만으로 도감을 해금하지 않는다.
- 도감 일러스트는 조과 사진과 분리해서 관리한다.
- 도감 일러스트를 앱에 번들할지, Supabase Storage에서 불러올지는 이미지 최적화 후 결정한다.
- 잡지 못한 어종은 잠김 상태로 보여줄 수 있다.
- 어종명이 직접 입력된 경우 기존 `fish_species`와 매칭할 수 없으면 임시 custom species로 처리할지 정책을 정한다.
- 일러스트 이미지의 출처와 라이선스는 구현 전에 확인한다.

## 완료 후 이동

조과 등록 성공 후 이동은 아래 중 하나를 선택한다.

- 방금 등록한 조과 상세 화면으로 이동
- 내 조과 목록으로 이동하고 목록을 갱신

1차 MVP에서는 사용자가 등록 결과를 확인하기 쉬운 상세 화면 이동을 우선 검토한다.

## 구현 시 확인할 것

- 로그인하지 않은 사용자는 조과 등록과 AI 판별 저장 흐름에 접근할 수 없다.
- 각 step에서 뒤로가기를 했을 때 입력값이 유지된다.
- 마지막 저장 전 필수값 검증이 동작한다.
- 조과 등록 성공 후 내 조과 목록 query가 갱신된다.
- 이미지 업로드, AI 판별, 위치 선택에는 loading/error 상태가 있다.
- 위치 권한 거부, 이미지 선택 취소, AI 판별 실패 케이스를 처리한다.
