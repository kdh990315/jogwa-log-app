# MVP 범위

## 1차 MVP 방향

1차 MVP는 "내 조과를 정확히 등록하는 것"에 집중한다. 통계 화면은 1차 MVP의 핵심 구현 범위가 아니지만, 1차 MVP의 데이터 구조는 나중에 통계를 낼 수 있도록 설계해야 한다.

## 1차 MVP 기능

- Auth: 사용자는 카카오, 애플, 구글 계정으로 로그인한다.
- Catch Log Write: 사용자는 조과를 직접 등록하거나 사진으로 등록한다.
- Catch Log Entry Mode: 사용자가 조과 등록 `+` 버튼을 누르면 "사진으로 등록"과 "직접 입력" 중 하나를 선택한다.
- Photo-based Catch Log: 사용자는 조과 사진 1장을 선택하거나 촬영해 어종, 촬영일시, 위치, 물때/날씨/바다낚시지수 정보가 채워진 조과 등록 초안을 만든다.
- Fish Species: 사용자는 어종을 직접 입력하거나 AI 어종 판별 결과를 선택해 기록한다.
- Catch Count: 사용자는 잡은 마릿수를 입력한다.
- Tide: 사용자는 물때를 선택하거나 입력한다. 예: 무시, 조금, 1물, 2물, 3물, 4물 등
- Weather: 사용자는 출조 당시 날씨를 선택하거나 입력한다.
- Location: 사용자는 카카오지도 기반으로 위치를 선택해 기록한다.
- Image Upload: 사용자는 조과 이미지를 업로드한다.
- AI Species Detection: 사용자는 이미지를 업로드해 Gemini 3.1 AI 기반 어종 판별 결과를 받는다.
- AI Fishing Advice: 사용자는 바다낚시지수와 어장관리 데이터를 바탕으로 AI 출조 상담 결과를 받는다.
- My Catch Logs: 사용자는 자신이 등록한 조과 목록을 확인한다.
- Catch Log Detail: 사용자는 조과 상세 정보를 확인한다.
- Species Dex: 사용자는 자신이 잡은 어종을 도감 형태로 확인하고, 잡은 어종의 일러스트 이미지를 볼 수 있다.
- Safety Reminder: 사용자는 낚시 중 철수 시간을 놓치지 않도록 30분 뒤, 1시간 뒤, 2시간 뒤, 직접 선택 방식으로 로컬 알림을 설정한다.
- Monetization Prep: 1차 MVP에서는 광고/제휴 수익화를 고려한 위치와 정책만 설계하고, 실제 광고 노출은 사용자 경험을 해치지 않는 범위에서 적용한다.

## 조과 등록 기본 데이터

1차 MVP에서 조과 등록 시 우선 저장해야 하는 데이터:

- 사용자 id
- 출조 날짜
- 어종
- 마릿수
- 물때
- 날씨
- 위치 이름
- 위도, 경도
- 조과 이미지 URL
- AI 어종 판별 결과
- 사용자가 최종 선택한 어종
- 도감 해금에 사용할 어종 id
- 조과 등록 방식: 직접 입력, 사진으로 등록
- 촬영 일시와 출처: 사진 EXIF, 기기 현재 시간, 사용자 직접 입력
- 위치 출처: 사진 EXIF, 현재 GPS, 지도 직접 선택, 사용자 직접 입력
- 자동 보강된 환경 데이터 출처: 바다낚시지수, 날씨 API, 수동 입력 등
- 사용자가 AI/자동 입력 결과를 수정했는지 여부
- 메모

## 사진으로 등록 MVP 범위

1차 MVP의 "사진으로 등록"은 사진 한 장으로 조과 기록 초안을 만들고, 사용자가 확인/수정 후 저장하는 흐름으로 시작한다.

상세 작업 문서: `doc/jogwalog-agent/photo-catch-log.md`

포함 범위:

- 조과 등록 `+` 진입점에서 "사진으로 등록"과 "직접 입력"을 선택할 수 있다.
- "사진으로 등록" 버튼명은 앱 안에서 우선 사용하고, 설명 문구에서 자동으로 채워진다는 점을 안내한다.
- 사용자는 사진 1장을 촬영하거나 앨범에서 선택한다.
- 앱은 사진으로 Gemini 3.1 AI 어종 판별을 요청하고 후보를 confidence와 함께 보여준다.
- 앱은 사진 EXIF에서 촬영일시와 위치를 가져올 수 있으면 조과 등록 초안에 반영한다.
- 사진에 위치가 없으면 현재 GPS 또는 지도 직접 선택으로 이어진다.
- 사진에 촬영일시가 없으면 기기 현재 시간 또는 사용자가 선택한 날짜/시간을 사용한다.
- 바다낚시 조과인 경우 위치와 날짜/오전·오후 기준으로 저장된 바다낚시지수 데이터를 매칭한다.
- 매칭 가능한 경우 물때, 파고, 수온, 기온, 유속, 풍속, 바다낚시지수 값을 초안에 자동 반영한다.
- 사용자는 어종, 마릿수, 크기, 위치, 날짜, 물때, 날씨, 메모를 저장 전 수정할 수 있다.
- AI 판별 결과와 자동 보강 데이터는 자동 확정하지 않고 사용자가 최종 확인한다.
- 자동 보강 데이터 조회에 실패해도 조과 기록 저장 자체는 막지 않는다.

제외 범위:

- 사진 여러 장을 한 번에 분석해 여러 조과를 자동 생성하는 기능
- 사진 속 여러 마리를 자동으로 세는 기능
- AI가 크기와 무게를 자동 확정하는 기능
- EXIF가 없는 사진의 과거 촬영 위치를 자동으로 추정하는 기능
- 모든 날씨/조석/조류 데이터를 정밀 시계열로 역추적하는 기능
- AI 판별 결과만으로 도감을 자동 해금하는 기능

주의:

- 사진으로 등록은 "자동 저장"이 아니라 "자동 초안 생성"으로 표현한다.
- 사용자의 정확한 포인트 위치는 기본적으로 비공개로 저장하고, 공유 기능에서는 좌표 흐림 또는 위치 숨김을 우선한다.
- AI 어종 판별, EXIF, GPS, 예보 매칭 결과에는 오류 가능성이 있으므로 저장 전 수정 가능해야 한다.
- Gemini API key와 외부 API key는 앱 코드에 넣지 않고 Supabase Edge Function secret으로 관리한다.
- 원본 사진, AI 판별 raw response, 자동 보강 데이터 출처는 나중에 재처리할 수 있도록 분리 저장한다.

## 향후 통계를 위한 데이터

향후 통계를 위해 가능하면 저장해야 하는 데이터:

- 출조 성공 여부
- 대표 어종
- 최대어 길이
- 최대어 무게
- 낚시 방식
- 포인트 타입
- 수온
- 풍속
- 기압
- 조류 또는 조위 정보
- 어종 도감 해금 여부
- 어종별 누적 조과 수
- 어종별 최대 사이즈

## 어종 도감 MVP 범위

1차 MVP의 도감은 전체 어종 백과사전이 아니라 "내가 잡은 어종 도감"으로 시작한다.

포함 범위:

- 사용자가 잡은 어종 목록 표시
- 잡은 어종의 일러스트 이미지 표시
- 아직 잡지 못한 어종은 잠김 상태로 표시
- 어종별 내가 잡은 횟수 표시
- 어종 상세에서 최근 조과 또는 대표 조과로 이동

제외 범위:

- 전체 어종 생태 백과
- 고급 필터/검색
- 지역별 출현 어종 추천
- 도감 업적/뱃지 시스템
- 일러스트 자동 생성

주의: 도감 일러스트는 AI 판별 이미지와 별도 개념이다. 사용자가 업로드한 조과 사진은 `catch_images`, 도감용 대표 일러스트는 `species_illustrations` 또는 `fish_species.illustration_url`처럼 분리해서 관리한다.

## 철수 알림 MVP 범위

1차 MVP의 철수 알림은 물때 자동 계산이 아니라 사용자가 직접 선택한 시간에 울리는 단순 로컬 알림으로 시작한다.

포함 범위:

- 기능명은 "철수 알림"으로 한다.
- 사용자는 30분 뒤, 1시간 뒤, 2시간 뒤, 직접 선택 중 하나로 알림 시간을 설정한다.
- 알림 문구는 안전한 이동을 유도하되, 생명안전용 알람처럼 보장되는 표현은 사용하지 않는다.
- 예약된 철수 알림이 있으면 앱 안에서 예정 시간을 확인하고 취소할 수 있다.
- 알림 권한이 거부된 경우 권한이 필요하다는 상태와 설정 안내를 보여준다.
- 구현은 기기 로컬 알림을 우선 사용하고, 서버 푸시나 Supabase 저장은 MVP 이후 확장으로 본다.

제외 범위:

- 간조/만조, 초들물/끝날물 등 물때 기반 자동 알림 시간 계산
- 현재 위치 기준 조석 관측소 자동 매칭
- 서버에서 알림을 예약하거나 여러 기기에 동기화하는 기능
- 방해금지, 무음, 배터리 최적화 상태에서도 반드시 울린다고 보장하는 표현

주의:

- 철수 알림은 안전을 돕는 보조 기능이며, OS 알림 권한과 기기 설정에 따라 울리지 않을 수 있음을 적절히 안내한다.
- 철수 알림은 조과 등록 데이터와 분리해서 설계하고, 조과 기록 저장을 방해하지 않아야 한다.

## AI 출조 상담 MVP 범위

1차 MVP의 AI 출조 상담은 정밀 예측이 아니라 "공식 바다낚시지수와 어장관리 데이터를 기반으로 한 반나절 단위 참고 상담"으로 시작한다.

포함 범위:

- 제목은 "AI에게 물어보세요" 또는 "AI 출조 상담"으로 한다.
- 사용자는 권역, 날짜, 오전/오후, 대상어종, 낚시방법을 선택한다.
- 날짜 선택 범위는 바다낚시지수 API가 제공하는 오늘 포함 7일 이내로 제한한다.
- 권역은 실제 포인트 전체가 아니라 대표 바다낚시지수 포인트와 매핑된 지역 단위로 제공한다. 예: 영흥도권, 태안권, 여수권, 통영권 등
- 바다낚시지수 API에서 위치, 날짜, 오전/오후, 물때, 파고, 수온, 기온, 유속, 풍속, 바다낚시지수를 가져온다.
- 어장관리 DB에서 선택한 권역, 어종, 날짜 또는 유사 조건의 최근 조과 패턴을 참고한다.
- 결과 화면은 "반나절 요약"과 "AI 출조 의견"을 기본으로 제공한다.
- 데이터가 부족하면 AI가 단정하지 않고 "참고할 조과 데이터가 부족하다"는 상태를 명확히 보여준다.

제외 범위:

- 15:00~19:00처럼 사용자가 지정한 세부 시간대별 정밀 분석
- 초들물, 중들물, 끝들물, 정조, 초날물, 중날물, 끝날물 자동 계산
- 기상청 단기예보, 조석예보 시계열, 조류예보 시계열을 모두 결합한 정밀 예측
- 정확한 포인트 단위 추천 또는 특정 장소의 조과 보장 표현
- AI가 외부 API를 직접 호출하거나 앱 코드에 API key를 보관하는 방식

주의:

- AI 출조 상담은 "예측"이나 "보장"이 아니라 참고용 분석으로 표현한다.
- 바다낚시지수, Gemini API key, 기타 외부 API key는 앱 코드에 넣지 않고 Supabase Edge Function secret으로 관리한다.
- 사용자의 정확한 조과 위치와 개인 식별 정보는 AI prompt에 직접 포함하지 않는다.
- 권역 기준 예보이므로 실제 포인트의 바람, 수심, 조류와 다를 수 있다는 안내를 결과에 포함한다.

## 제외 범위

1차 MVP 제외 범위:

- 고급 통계 대시보드
- 물때별/날씨별/어종별 차트
- 연간 출조 횟수와 성공률 분석 화면
- 최대어 랭킹
- 커뮤니티 피드
- 좋아요/댓글
- 팔로우
- 지도 기반 포인트 공유
- 실시간 채팅
- 결제
- 관리자 페이지
- 오프라인 모드
- 물때 기반 자동 철수 알림
- AI 출조 상담의 시간대별 정밀 물흐름 분석
- 과도한 전면 광고
- 조과 등록 흐름을 방해하는 광고
- 사진으로 등록에서 AI가 판단한 값을 사용자 확인 없이 자동 저장하는 흐름

주의: 통계 화면은 1차 MVP에서 제외하지만, 통계를 만들 수 없게 데이터를 대충 저장하면 안 된다. 조과 등록 데이터는 향후 통계 확장을 전제로 구조화해서 저장한다.

## 프로젝트별 결정사항

- 1차 MVP는 조과 등록 품질에 집중한다.
- 통계 화면은 1차 MVP에서 제외하지만 통계가 가능한 데이터 구조를 먼저 만든다.
- 조과 이미지는 MVP에서 최소 1장 이상 업로드할 수 있게 한다. 최대 장수는 추후 결정한다.
- 조과 등록 진입점은 "사진으로 등록"과 "직접 입력" 두 가지로 제공한다.
- "사진으로 등록"은 사진 기반 자동 초안 생성 후 사용자가 확인/수정해 저장하는 흐름으로 구현한다.
- 물때와 날씨는 직접 입력을 지원하되, 사진으로 등록에서는 가능한 경우 바다낚시지수/날씨 데이터로 자동 보강한다.
- 위치는 카카오지도 기반으로 선택하고 위치 이름, 위도, 경도를 저장한다.
- AI 어종 판별 결과는 자동 확정하지 않는다. 사용자가 최종 선택하거나 수정할 수 있어야 한다.
- AI 출조 상담 v1은 바다낚시지수 기반 반나절 상담으로 시작하고, 시간대별 날씨/조석/조류 분석은 MVP 이후로 분리한다.
- AI 출조 상담에 사용하는 외부 데이터는 Supabase Edge Function에서 조회하고, 앱은 Edge Function 결과만 사용한다.
- 철수 알림은 30분 뒤, 1시간 뒤, 2시간 뒤, 직접 선택 방식의 로컬 알림으로 시작한다.
- 철수 알림의 물때 자동 계산과 서버 동기화는 MVP 이후에 검토한다.
- 도감은 1차 MVP에서 "내가 잡은 어종 + 일러스트 이미지" 중심으로 구현한다.
- 도감 일러스트를 앱에 번들할지, Supabase Storage에서 불러올지는 이미지 최적화 후 결정한다.
- 도감 일러스트 데이터의 출처와 저작권은 구현 전에 확인한다.
- 1차 MVP는 라이트모드와 다크모드를 모두 지원한다.
- 사용자가 테마를 직접 선택하지 않았다면 시간대 기준으로 라이트/다크모드를 자동 적용한다.
- 앱 광고는 Google AdMob 기준으로 검토한다. 모바일 앱에서 Google AdSense 코드를 직접 넣는 방식은 우선하지 않는다.
- 쿠팡 수익화는 쿠팡 파트너스 제휴 링크/배너 기준으로 검토한다.
- 광고와 제휴 링크는 조과 등록 흐름을 방해하지 않게 배치한다.
- 1차 MVP 완료 후 iOS App Store와 Samsung Galaxy Store 배포를 목표로 한다.
- 앱 분석은 Google Analytics for Firebase를 우선 검토한다.
- 조과 등록 funnel, AI 판별 사용량, 도감 조회 이벤트를 우선 설계한다.
- Gemini 3.1 API 세부 스펙은 구현 시점에 공식 문서로 확인한다.
- 커뮤니티 기능은 MVP 이후에 별도 단계로 검토한다.
